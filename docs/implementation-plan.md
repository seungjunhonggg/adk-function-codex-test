# 구현 계획 (v0)

## 전제 (PLAN.md)
- 시뮬레이션 워크플로우가 최우선.
- chip_type 입력 시 1-2를 생략하고 1-3부터 시작.
- 브리핑은 한글 라벨만 사용(영문 컬럼명 금지).
- 모델 컨텍스트 한도는 10k, 요약은 컴팩트하게 유지.
- DB 없이도 데모 모드로 테스트 가능해야 함.

## 단계
### Phase 1: 산출물 정의 (완료)
- 브리핑 산출물 스펙: `docs/briefing-output-spec.md`

### Phase 2: 데이터 계약 (완료)
- 단계별 데이터 계약: `docs/data-contracts.md`

### Phase 3: 워크플로우 + 라우팅 (다음)
- 캐주얼 vs 시뮬레이션 라우팅 정의.
- 중간 단계 변경 시 무효화 규칙 정의.
- /api/chat 워크플로우 문서 정리(완성 시 반영).

### Phase 4: 툴 + MCP 설계
- 최소 툴셋 정의(DB 조회, 최적화 API, 리포트 생성).
- 외부 서비스 사용 시 MCP 경계 정의.

### Phase 5: 메모리 + 컨텍스트 전략
- 세션 메모리 스키마(inputs/outputs/status).
- 컨텍스트 예산 및 컴팩션 규칙 정의.

### Phase 6: 데모 모드
- 데모 픽스처/생성기 구축.
- 실제 스키마와 일치성 검증.

## 남은 입력
- 칩기종 테이블 스키마.
- API 응답 예시.
- 설계 파라미터 8~10개 실제 컬럼명.
- 불량률 지표 목록.
